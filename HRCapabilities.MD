## Onboarding and Offboarding
 This is for existing organization in cyrev who want to manage contracting business (business needs to be registered before members of the contacting organization can be onboarded on cyrev)
#### To register a new firm 
   1. The relationship manager will have to approach the admin to create the firm.
   2. When the business has been registered, the relationship manager can then proceed to create the members of the business (business_users), 
      this basically creates the users in entra in pending state and then notify the admin for approval
   3. When step 3 is done, the admin gets a notifications
   4. The admin then decides to approve or deny such an approval request.

Create a data model with two tables (business and business_users) the business has the following fields: 
    
``` json 
        id (autogenerated), 
        org code, 
        company name, 
        contract start date, 
        contract end date, 
        relationship owner, 
        contract status, 
        employeeID format (a combination of ORG code and serially generated number), etc.
   ```

the business_users has the following fields 

```json 
    firstname, 
    lastname, 
    employeeID {autogenerated}, 
    company {entity company name}, 
    manager, 
    start-date, 
    end-date, 
    unit, 
    department, 
    division, 
    identity status {lifecycle states}, etc
```
